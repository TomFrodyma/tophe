# Creating the "Deploy on Railway" template

One-time, ~10 minutes, done in the Railway dashboard (it's tied to your
account, so only you can do this part). Do it **after** the fresh public repo
exists — the template points at it.

## 1. Open the template composer

Railway dashboard → **New** → **Template** (or railway.com/compose).

## 2. Service: `app`

- **Source:** GitHub repo `TomFrodyma/tophe` (the public one), branch `main`.
  The repo's `railway.json` selects the Dockerfile build; the image syncs the
  database schema itself on every boot (additive-only — destructive changes
  refuse and wait for a human).
- **Networking:** enable a public domain.
- **Volume:** mount at `/data`.
- **Variables:**

  | Variable | Value |
  | --- | --- |
  | `DATABASE_URL` | `${{Postgres.DATABASE_URL}}?sslmode=require&uselibpqcompat=true` |
  | `BETTER_AUTH_SECRET` | `${{secret(32)}}` |
  | `CALENDAR_INTEGRATION_SECRET` | `${{secret(32)}}` |
  | `CRON_SECRET` | `${{secret(32)}}` |
  | `NEXT_PUBLIC_SAAS_URL` | `https://${{RAILWAY_PUBLIC_DOMAIN}}` |
  | `STORAGE_VOLUME_PATH` | `/data` |
  | `ANTHROPIC_API_KEY` | leave empty, mark **optional** with the description "Optional: enables the Claude AI models. Get a key at console.anthropic.com" |
| `OPENAI_API_KEY` | leave empty, mark **optional** with the description "Optional: enables OpenAI - or any OpenAI-compatible provider (OpenRouter, Groq, ...) by also setting OPENAI_BASE_URL" |

  Web Push keys are generated by the app itself — nothing to add. Avatars
  (S3) are deliberately left out of the template.

## 3. Service: `Postgres`

Add Railway's official **PostgreSQL** database service. Nothing to configure.

## 4. Service: `cron`

- **Source:** Docker image `alpine:latest`
- **Cron schedule:** `*/5 * * * *`
- **Custom start command:**

  ```
  sh -c 'apk add --no-cache curl >/dev/null && for p in daily-reminder calendar-reminders briefing; do curl -fsS -m 60 -X POST -H "Authorization: Bearer $CRON_SECRET" "$SAAS_URL/api/cron/$p" || true; done'
  ```

- **Variables:**

  | Variable | Value |
  | --- | --- |
  | `CRON_SECRET` | `${{app.CRON_SECRET}}` |
  | `SAAS_URL` | `https://${{app.RAILWAY_PUBLIC_DOMAIN}}` |

## 5. Publish and test

1. Publish the template; Railway gives you a template URL.
2. **Deploy it yourself once from the button** before sharing: open the app's
   domain, create the account (first-run screen), enable browser
   notifications in Settings, and send the test notification.
3. Check the login page redirects use the right domain — that confirms
   `NEXT_PUBLIC_SAAS_URL` reached the Docker build (Railway passes service
   variables as build args for declared `ARG`s).

## 6. Add the button to the README

```markdown
[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/template/YOUR-TEMPLATE-CODE)
```

Bonus: published templates are part of Railway's kickback program — you earn
a share of the usage they generate.
